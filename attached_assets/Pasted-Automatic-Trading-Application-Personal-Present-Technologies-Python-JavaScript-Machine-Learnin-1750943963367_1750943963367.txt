Python, JavaScript ,Machine Learning (ANN, LSTM), Jupyter Notebook,TensorFlow 2.0, SKlearn and Selineum

Key Processes:
Data, feature preparation and training the ML model:

Automatic Digitization of Seismic shot gather scans (2017)
For EyeCube solutions built a tool for Automatic Digitization of hard copy scan of Seismic shot gathers. Where it uses Machine Learning to track the signal on the image.
Technologies:
Matlab, Machine Learning, CV2
Critical aspects:
Image processing, Identification of start of signal line, and identification of next point on the next row of pixels with machine learning
Key Processes:
Processing the Image:
Every image contains the long signals of 24 arrays. At places neighbouring signal merge and it becomes complicated to predict signal. There is noise on the image that are reduced with image filtering.
Feature preparation and training the ML model: 
A ML model devised by training with manually picking few signals on images.
Applied the model to pick the signals on lot of images, by searching on next adjacent row. The pixels to be chosen on next row is predefined. It is- equal number of pixels on both sides of current predicted point. The ML model predict the placement of next successful pixel relative to current pixel.
Outcome: 
Matlab executable application which can run on any Windows machine and digitize the seismic signals

Prediction of Oil producing zones in an Oil Field (Kharsang-India, 2016-17)
Technologies:
Matlab, Machine Learning (Classification & Decision trees)
Critical aspects:
Cleaning and preparation of data, Training the model
Key Processes:
Data, feature preparation and training the ML model: 
Well logs are electronic logs which record a continuous signal against the formation in a well. Usually producers record 8 to 10 useful logs which record different kind of responses from the formation (Resistivity, Gamma ray log, Neutron porosity, electron density, Static Potential etc.). This data usually comes one at each foot.
There are problems in the data, i.e, like missing data, erroneous data etc. Processed to handle these problems.
The outcome against each sample would the Oil saturation and produceability.
Prepared around 700 samples in a field, each representing 1 meter.
Machine Learning model:
Two models had been tested, one single classification tree and the other is bagged decision tree. The bagged decision tree gave good result
Outcome:
Client was provided with new zones which can add reserves around 2MMBBL
